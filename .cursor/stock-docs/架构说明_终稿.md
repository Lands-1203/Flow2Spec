# Flow2Spec（@double-codeing/flow2spec）架构说明

> 由 `stock-docs/架构说明_初稿.md` 经 **f2s-doc-final** 归纳；侧重包内职责、init 机制与配置根约定，供 **f2s-ctx-build** 消费。

---

## 核心概念

| 概念 | 说明 |
|------|------|
| **Flow2Spec** | npm 包 `@double-codeing/flow2spec`，可执行名 `flow2spec`；在业务仓库根执行 init，向 AI 工具配置目录落盘规则、技能与模版。 |
| **配置根** | 各 AI 工具约定的项目内目录，如 `.cursor`、`.claude`、`.codex`；其下含 `rules/`、`skills/`、`template/`、预建 `stock-docs/`、`req-docs/` 等。 |
| **配置根的父目录** | 用户执行 `flow2spec init` 时的当前工作目录（`process.cwd()`），即业务代码仓库根。 |
| **agent** | `cursor` / `claude` / `codex`，对应不同配置根目录名；init 可单选或多选，默认仅 `cursor`。 |
| **templates/** | 包内**权威模板源**，随 `files` 发布；init 时递归复制到各选中配置根（同名路径覆盖）。 |
| **SUBDIRS** | init 时确保存在的子目录：`stock-docs`、`req-docs`、`template`、`rules`、`skills`。 |
| **本包运行时边界** | CLI 解析 + 文件复制；无 HTTP 服务、无数据库、无对业务源码目录的结构性改写（除配置根下文件）。 |

---

## 状态与流转

本说明**无业务状态机**。可理解为主流程上的阶段：

- **未 init**：业务仓库无（或不全）Flow2Spec 约定目录与模板内容。
- **已 init**：配置根下已具备 `rules`、`skills`、`template` 及空/可用的 `stock-docs`、`req-docs`；后续由团队在对话中按 Skill 演进，与本包版本升级可再次 init 覆盖模板。

---

## 业务规则

- **Node 版本**：运行环境与发布对象需满足 `engines.node`：**>= 16**。
- **init 参数**：`init` 后的参数为 agent 列表；**省略时等价于仅 `cursor`**。未知 agent 须报错并提示合法取值（`lib/agents.js` 中 `AGENTS` 的 key）。
- **多 agent**：同一 agent 在一条命令中重复出现时**去重**；合法 id 大小写不敏感，支持去掉前导 `--` 的形式。
- **复制策略**：`templates/rules`、`templates/skills`、`templates/template` 下文件**已存在则覆盖**；`ensureDirs` 仅保证目录存在，不删除用户已在 `stock-docs`/`req-docs` 中自行创建的文件。
- **发布包内容**：以 `package.json` 的 `files` 为准（`cli.js`、`lib`、`templates`、`README.md`、`docs`）；未列入者不会进入 npm 包。
- **质量门禁**：`npm test` 与 `prepublishOnly` 执行 `node cli.js --help`；`pack:check` 使用 `npm pack --dry-run` 校验包清单。

---

## 关键流程

1. **查看帮助**：执行 `flow2spec --help`、`-h` 或无子命令时输出用法与 agent 列表；进程退出码 `0`。
2. **初始化配置根**：在配置根的父目录执行 `flow2spec init [agent ...]` → `cli.js` 调用 `lib/init.run(cwd, agentIds)` → 对每个 agent：`ensureDirs` + `copyTemplatesToAgentRoot`；成功则打印各配置根路径说明，失败则 `process.exit(1)` 并输出错误信息。
3. **消费者日常使用**：在 IDE 中按场景加载 `skills/<标识>/SKILL.md` 工作流；架构/需求文档按约定放入 `stock-docs/`、`req-docs/`，与业务代码并行维护（详见包内 `docs/` 与 README 导航）。

## 接口 / API / 页面（CLI 约定）

| 调用方式 | 行为 | 结果 |
|----------|------|------|
| `flow2spec`、`flow2spec --help`、`-h` | 打印帮助 | 标准输出帮助文本，退出码 `0` |
| `flow2spec init` | 默认 agent：`cursor` | 向 `./.cursor/` 写入模板与预建目录 |
| `flow2spec init <agent> [...]` | 指定一个或多个 agent | 向对应 `./.cursor/`、`./.claude/`、`./.codex/` 分别写入 |
| 其他子命令 | 未实现 | 打印帮助并以退出码 `1` 结束 |

**内部入口**：`require("./lib/init")` 的默认导出为异步函数 `run(cwd, agentIds)`，返回已处理的 agent id 数组。

---

## 配置 / 数据 / 错误

- **配置**：包元数据在 `package.json`（`name`、`version`、`bin`、`main`、`files`、`engines`）；无运行时外部配置文件依赖。
- **数据**：无持久化业务数据；仅文件系统读写。
- **错误**：未知 agent 时抛出 `Error`，消息包含「未知 agent」与可选列表；其他 I/O 异常由 Node 抛出并在 `cli.js` 的 `catch` 中打印。

## 实现位置与对接方式

| 内容 | 路径或说明 |
|------|------------|
| 进程入口 | `cli.js`（shebang `node`） |
| agent 与子目录常量 | `lib/agents.js`：`AGENTS`、`SUBDIRS`、`normalizeAgentIds` |
| init 实现 | `lib/init.js`：`ensureDirs`、`copyRecursive`、`copyTemplatesToAgentRoot`、`run` |
| 模板源 | 包内 `templates/rules`、`templates/skills`、`templates/template`（及预置 `stock-docs`、`req-docs` 空目录若存在） |
| 人类文档 | `README.md`、`docs/*.md` |
| **维护者对接** | 修改模板后 bump 版本并发布；使用方在业务仓库执行 `npx @double-codeing/flow2spec init` 或等价命令更新配置根。**templates/ 与本仓库 `.cursor/` 可能不同步**，维护时需约定以何者为源或定期对齐。 |
