# skill-authoring（路由摘要）

> 适用范围：在 **本仓**（Flow2Spec 产品开发仓）新增或重构 `f2s-*` SKILL 时遵循。**不下发**到下游业务仓——下游不写 `f2s-*` SKILL，本主题对其无意义。

## 作用

统一 `templates/skills/f2s-*/SKILL.md` 的小标题命名与排列顺序，避免同类信息出现多种写法（`## 输入` / `## 入参` / `## 传参`；`## 步骤` / `## 执行流程` / `## 强制流程`）造成跨 SKILL 维护心智成本。本主题**只规定命名与顺序**，不限制正文风格。

## SKILL 骨架（必填段标 ✱，可选段标 ◇）

```
---
name: f2s-<id>
description: <一句话 + 「；触发：<词1>、<词2>、<词3>」>
---

> 执行口径：<本技能维护什么 / 不动什么 / 是否绑配置开关；一段话>

# <主标题>

## 与既有技能的边界 ◇       ← 触发词高度近似时建议写
## 适用范围 ✱               ← 替代旧版「使用时机/何时使用/适用场景」
## 编排 ✱                   ← 主 / 子 agent 切分；指向 unified-entry 取 subAgent / switchAgentVerification 语义
## 输入 ✱                   ← 替代旧版「入参/传参」
## 强制前置 ◇               ← 本技能在步骤前必须 Read 的 rule / 配置文件
## 流程 ✱                   ← 替代旧版「步骤/执行步骤/执行流程/强制流程」
   ### 步骤 N: <动词短语>     ← 子标题统一「步骤 N」前缀；编号与正文顺序一致
## 输出摘要 ✱               ← 替代旧版「输出摘要格式/模板（建议）」
## 约束与禁止项 ✱           ← 合并旧版「约束 / 路径与引用约束 / 禁止项」
## 复杂场景示例 ◇           ← 易踩坑的技能写 1–3 个
## 完成后自检 ✱             ← 编号列表；每条对应可校验的事实
```

## 字段约定

### `description`

- 一句话说明做什么 + 半句话触发条件，末尾接 `；触发：<词1>、<词2>、<词3>`。
- 触发词与同名 `matchers/<id>.json` 的 `includeAny` 保持口径一致。
- 不写实现细节 / 模型选择 / 提示工程技巧。

### `执行口径`（顶部 blockquote）

- 一段话讲清三件事：**维护哪些路径** / **不动哪些路径** / **是否绑某个 `flow2spec.config.json` 字段**。
- 例：「本技能只维护 `.Knowledge`，不改配置根 `rules/skills`，不动业务代码，不创建 `.task/`。」

### `编排`

- `subAgent` / `switchAgentVerification` 语义统一指向 `f2s-flow2spec-unified-entry`，**不复述定义**——只写本技能在该字段为 `true` / `false` 时的差异化行为。
- 写权硬约束（哪些文件恒由主 agent 落盘）单独成一行。
- 验证策略（落盘侧自验 / 交叉校验）单独成一行。

### `流程`

- 子标题统一 `### 步骤 N: <动词短语>`，全文冒号风格统一。
- 编号从 1 起；`0` / `0.5` 是历史包袱，新 SKILL 避免使用——前置改放 `## 强制前置`。
- 每步开头一句"做什么"，再列约束 / 落盘文件 / 输出形态。

### `约束与禁止项`

- 列表式，肯定式优先（参见 `f2s-flow2spec-unified-entry`「知识库落盘文风」）。
- 排他性选择例外。
- 写权硬约束已在「编排」写过则不复述，本节仅列领域级禁止项（如"不动配置根 `rules/skills`"）。

### `完成后自检`

- 每条对应可校验的事实（agent 能用 `Read` / `Grep` 验证），避免"是否合理 / 是否充分"这类主观项。
- 命中本技能特有风险点（例：`f2s-kb-addRules` 必检"是否在用户未确认前提前落盘"）。

## 不规定的事

- **正文风格**：每个 SKILL 按场景需要写散文 / 表格 / 流程图，本主题不限定。
- **小节内部顺序**：例如「编排」节内各点的排列，按 SKILL 实际重要性排。
- **是否复述配置开关**：默认指向 unified-entry 不复述；若需强调某字段对自身的特殊影响（如 `f2s-doc-milestone` 不受 `flow2spec.config` 编排开关影响），可在「编排」节单独写一句。

## 参考实现

- 最近按本模板写的：`templates/skills/f2s-kb-addRules/SKILL.md`（含「与既有技能的边界」「强制前置」「步骤 1–5」「输出摘要」「约束」「复杂场景示例」「完成后自检」）。
- 风格接近的旧 SKILL：`templates/skills/f2s-kb-build/SKILL.md`（小标题用「步骤 N：xxx」而非子标题，下次维护时顺手降级即可）。

## 演进策略

- **新 SKILL**：必须按本模板写。
- **存量 SKILL**：不强制批量改名；下次维护到某个 SKILL 时顺手对齐到本模板。
- **离群严重**（`f2s-doc-arch` / `f2s-doc-final` / `f2s-kb-merge` / `f2s-req-clarify` 等缺多个必填段）单独立项重构，不与日常维护混在一起。
