# f2s-kb-distill：Q&A 驱动知识提取与入库

Q&A 驱动知识提取技能，从普通问答中自动萃取可复用知识事实并入库；根据下钻深度与描述层次自动判断创建新 topic 或追加到现有 topic。

## 适用场景

- 用户提问 → Agent 下钻源码回答 → 需要将发现的知识固化到知识库
- 通常由 `f2s-kb-feedback-closing` 规则自动建议，也可用户手动调用
- 与 `f2s-kb-sync` 区别：`sync` 用于批量同步多个能力；`distill` 专注单次 Q&A 知识提取

## 执行前依赖

本技能执行前须先读以下主题：

- **`f2s-kb-feedback-closing`**：定义"可复用知识事实"边界（核心机制、状态流转、契约、配置影响、失败回退、模块边界、数据模型）
- **`f2s-topic-authoring`**：创建/修改 topic 时的命名、骨架、metadata、dependencies 准则

## 核心流程

### 输入（自动提取）

- 用户问题：从对话历史自动提取
- Agent 回答：完整回答内容
- 已匹配 topic：若由 `f2s-kb-feedback-closing` 触发则携带
- 下钻文件：从回答中提取引用的文件路径、函数名、行号

### 判断逻辑

**下钻深度评分**（0-50 分，累加）：
- 读文件数：0 / 1-2 / 3-5 / 6+ → 0 / 3 / 7 / 10 分
- 分段读次数：0-1 / 2-4 / 5-8 / 9+ → 0 / 3 / 7 / 10 分
- 函数引用数：0-2 / 3-5 / 6-10 / 11+ → 0 / 3 / 7 / 10 分
- 代码片段长度：0-50 / 51-150 / 151-300 / 301+ 行 → 0 / 3 / 7 / 10 分
- 回答长度：0-200 / 201-500 / 501-1000 / 1001+ 字符 → 0 / 3 / 7 / 10 分

**深度分类**：
- **浅层**（0-15 分）：简单问答，源码引用少
- **中层**（16-30 分）：中等复杂度，多文件查阅
- **深层**（31-50 分）：深入探索，大量源码分析

**知识描述层次**（按内容特征判断，非长度）：
- **摘要级**：仅结论性描述（"是什么"/"做什么"），无条件/流程/函数细节
- **详细级**：含机制解释、流程步骤、关键判断条件（"当 X"、"若 Y 则"、"首先...然后..."）
- **实现级**：含函数调用关系、状态转换细节、边界条件处理、代码示例

**策略决策矩阵**：

| 下钻深度 | 已匹配 topic | 现有 topic 描述层次 | 提取知识描述层次 | 策略 |
|---------|-------------|-------------------|-----------------|------|
| 浅层 | 匹配 | 摘要 | 摘要 | **追加到现有** |
| 浅层 | 匹配 | 摘要/详细 | 详细 | **追加到现有** |
| 浅层 | 匹配 | 摘要 | 实现 | **创建子 topic** |
| 浅层 | 未匹配 | - | 任意 | **创建新 topic** |
| 中层 | 匹配 | 摘要 | 摘要/详细 | **追加到现有** |
| 中层 | 匹配 | 摘要 | 实现 | **创建子 topic** |
| 中层 | 匹配 | 详细/实现 | 详细/实现 | **追加到现有** |
| 中层 | 未匹配 | - | 任意 | **创建新 topic** |
| 深层 | 匹配 | 摘要 | 任意 | **创建子 topic** |
| 深层 | 匹配 | 详细/实现 | 详细/实现 | **追加现有或创建子 topic** |
| 深层 | 未匹配 | - | 任意 | **创建独立模块 topic** |

**决策关键**：
- 描述层次差距 ≥ 2 级（摘要 vs 实现）→ 强制创建子 topic
- 描述层次差距 = 1 级（摘要 vs 详细，或详细 vs 实现）→ 可追加
- 描述层次匹配（同级）→ 正常追加
- 深层下钻 ≥ 深层 → 优先创建子 topic

### 落盘与路由同步

- **追加到现有 topic**：读取 2-3 个近邻 topic 样本对齐风格，在相关段落后追加内容
- **创建子 topic**：生成新 topicId（基于父 topic + 焦点），更新父 topic 添加链接，写入 `topicDependencies`
- **创建独立模块 topic**：深层下钻时可选创建 stock-doc，topic 作为摘要 + 指针
- **同步 manifest**：新 topic 写入 `topicPaths`、`taskToTopicRules`、matcher 分片；已有 topic 检查是否需补 `includeAny`
- **同步 index.md**：新 topic 添加条目，格式保持一致

### topicMetadata 处理

有明确证据时写入：
- 核心机制/状态流转/失败回退 → `policy`
- 配置开关影响 → `config`
- 模块边界/调用约定 → `module`
- 已实现能力/业务逻辑 → `feature`
- `confidence` 设为 `inferred`
- 无明确证据时不写，摘要中列为"待分类"

## 边界与禁止项

- 只维护 `.Knowledge`，不改配置根 `rules/skills`
- 无需用户确认（Q&A 已验证知识正确性）
- 保持轻量，单次 Q&A 知识提取 30 秒内完成
- 避免过度拆分：除非下钻深度 ≥ 深层且知识描述层次 ≥ 详细级，优先追加到现有 topic
- 生成的 matcher `includeAny` 应覆盖用户实际使用的表达，不仅是技术术语
